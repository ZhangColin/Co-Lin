## 📖 项目简介

**Project Co-Lin** 是基于DDD思想构建的个人知识操作系统，也是Colin的数字分身训练场。

20年一线开发经验，多次尝试积累都不了了之。这次不一样：
- Colin（人）+ Colin（AI分身）双核驱动
- 实践驱动，边做边总结，持续沉淀
- 打破"计划优化陷阱"，打破"不了了之"

## 🎯 核心目标

1. **打破"不了了之"**：多年来的积累尝试总是半途而废，这次要形成持续沉淀的正向飞轮
2. **实践驱动积累**：不追求完美规划，边做边总结，通用经验沉淀到这里
3. **训练数字分身**：通过持续对话和文档积累，让Colin AI越来越像Colin本人

## 🏗 目录架构

### 1. 🧠 Core_Kernel (核心内核)
操作系统底层，决定"我是谁"和"如何做决定"。

* **1.1 Principles/** - 原则和方法论（第一性原理、奥卡姆剃刀、DDD等）
* **1.2 Bio_Data/** - 个人档案、背景、优势劣势分析
* **1.3 Vision_Goals/** - 长期目标、阶段性OKR

### 2. ⚡ Active_Process (运行进程)
流动性最强，正在处理的任务和正在输入的知识。

* **2.1 Current_Projects/** - 进行中的项目
* **2.2 Learning_Buffer/** - 正在学的内容、待理解的概念
* **2.3 Brainstorming/** - 未成型的想法、对话记录、灵感碎片

### 3. 💾 Knowledge_Base (知识库)
经过验证、可复用的沉淀。

* **3.1 Industry_Methodology/** - 行业方法论（DDD、敏捷、TDD等）
* **3.2 Experience_Log/** - 经验复盘、路径记录
* **3.3 Output_Archive/** - 完成的作品、文档、方案

### 4. 🤖 AI_Collaboration (AI协作)
与Colin AI分身的协作界面。

* **4.1 Prompts_Library/** - 提示词库、有效的指令模板
* **4.2 Colin_Training/** - 分身训练素材（行文风格、思维模式）
* **4.3 Feedback_Loop/** - 纠偏反馈、持续改进记录


## 🛠 核心信条

这些原则贯穿所有协作：

**知行合一**
- 以大多数人的努力程度，根本轮不到拼天赋
- 不闻不若闻之，闻之不若见之，见之不若知之，知之不若行之

**方案思维**
- 提问题必带方案，没方案时明确声明
- 没有更好方案时，当前方案就是最好的，要勇于担责，要执行

**果断决策**
- 开放讨论，但讨论出发点必须是解决问题
- 反对为讨论而讨论，为反对而反对
- 讨论必须聚焦，必须有人拍板

**实践优先**
- 完成>完美，执行>规划
- 边做边调整，在行动中迭代
- 打破计划优化陷阱

## 🚀 工作流程

### 日常协作
1. **问题讨论** - 与Colin AI讨论技术方案、学习路径、工作决策
2. **实践执行** - 在真实项目中实践，不追求完美规划
3. **经验提炼** - 讨论后识别可沉淀内容
4. **文档沉淀** - 通用经验写到Co-Lin，项目特定留项目

### 分身训练
1. **对话积累** - 每次深度讨论都是训练素材
2. **文档学习** - Colin AI定期读取新增文档，持续学习
3. **风格同构** - 学习行文习惯和语气，越来越像Colin本人
4. **能力进化** - 形成正向飞轮：实践→沉淀→学习→更强

### 内容组织原则
- **通用经验** - 沉淀到Co-Lin（方法论、路径、复盘）
- **项目特定** - 留在各项目中（具体实现、业务细节）
- **判断标准** - 抽离项目特定信息后仍有价值的才放Co-Lin
